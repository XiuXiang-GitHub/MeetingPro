# ADR-004: 四 Tab 导航架构

**Status**: Accepted  
**Date**: 2026-05-25  
**Deciders**: Alan

## Context

小程序包含 6 大功能模块（首页仪表盘、摆台编辑器、人员调度、会议档期、物资管理、执行清单、数据报表），需要设计信息架构和导航方式。

## Decision

采用 4 Tab 底部导航 + "更多"页面作为二级入口。

| Tab | 页面 | 说明 |
|-----|------|------|
| 首页 | `pages/index/index` | 仪表盘聚合视图 |
| 摆台 | `pages/layout/editor/index` | 核心功能，直达编辑器 |
| 调度 | `pages/scheduling/calendar/index` | 排班周历 |
| 更多 | `pages/more/index` | 网格菜单，收纳其余功能 |

"更多"页面内含 5 个入口：会议档期、员工档案、物资管理、执行清单、数据报表。

## Rationale

- **微信规定**：tabBar 最多 5 项，4 项预留扩展空间
- **优先级排序**：首页和摆台是最高频操作，占据前两个 tab；调度是日常运营核心，第三个 tab
- **避免过载**：低频功能（报表、清单）收纳在"更多"中，保持主 tab 简洁
- **直接操作路径**：摆台 tab 直达编辑器而非历史列表，减少一次点击

## Consequences

- "更多"页面承担二级导航角色，需保持入口清晰
- 会议档期、物资管理等功能隐藏在二级页面，新用户可能不易发现
- 后续功能增加时可利用第 5 个 tab 位或扩展"更多"网格
