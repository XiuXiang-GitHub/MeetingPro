# ADR-003: Botanical Calm 设计系统

**Status**: Accepted  
**Date**: 2026-05-25  
**Deciders**: Alan

## Context

项目从微信云开发 quickstart 模板改造而来，原始 UI 为通用的功能列表式界面，与酒店会议管理的高端定位不匹配。需要建立完整的视觉设计语言。

## Decision

采用 "Botanical Calm" 设计系统，灵感参考 Aesop 门店、高端水疗。

**核心色彩**：
- 主色鼠尾草绿：`#8C9E7A`（primary）、`#6B7D5C`（dark）、`#A8B89A`（light）
- 深色森林夜：`#2C3328`（文字主色）
- 语义色：成功 `#5C9A6F`、警告 `#E09E4A`、危险 `#C5655E`
- 页面底色：`#FAF9F6`（暖白）
- 卡片：`#FFFFFF` + 微阴影

**设计原则**：低饱和、高呼吸感、克制中传递品质

**技术实现**：
- 全局样式定义在 `app.wxss`（卡片系统、按钮系统、标签系统、动画系统、布局工具）
- 页面特有样式在各自 `.wxss` 文件中
- 动画使用 CSS `@keyframes`（fadeUp、popIn、breathe）
- 不使用任何第三方 UI 库或 npm 包

## Rationale

- 酒店行业对美学要求高，通用 UI 无法满足
- 纯 CSS 实现，无额外依赖，符合微信小程序约束
- 色彩系统选自自然植物色调，与酒店环境协调
- 牺牲部分功能密度换取视觉品质

## Consequences

- 所有新页面必须遵循此设计系统
- 色彩/阴影/圆角必须使用系统变量，不得引入新的视觉风格
- 可访问性需额外关注：低饱和配色需确保文字对比度达标（当前 `#9EA395` 在白色背景上对比度约 2.8:1，低于 WCAG AA 4.5:1 标准）
